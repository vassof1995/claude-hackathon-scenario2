Scenario 2 · Cloud Migration

The Lift, the Shift, and the 4am Call

Team The 4am Club · Contoso Financial → AWS

Cloud-ready, runs locally: MinIO→S3 · Postgres→RDS · Redis→ElastiCache.

Built for three readers: the auditor reads the IaC, the CTO reads the ADRs, ops runs the runbook.

The Problem and the Tension

  • Three workloads: customer web app, nightly batch reconciliation, a reporting DB five teams query directly.
  • CFO signed the contract; CTO wants cloud-native, not lift-and-shift; compliance wants residency; SRE wants sleep.
  • Stakeholders are not aligned — and the docs hide the real coupling.

The Decision: One 6-R Does Not Fit All Three

  • Frontend: re-platform — built Vue SPA to S3 + CloudFront.
  • API: re-host — same Spring Boot image on ECS Fargate behind an ALB.
  • Batch: re-architect the trigger only — EventBridge Scheduler to a run-to-exit ECS task.
  • Reporting DB: re-platform — RDS Postgres (Multi-AZ) + read replica.
  • Cloud-native where it pays; risky refactors (caching, DB split) explicitly deferred.

What We Built

  • A compose stack mapped one-to-one to AWS primitives: MinIO to S3, Postgres to RDS, Redis to ElastiCache, containers to ECS behind an ALB.
  • Terraform IaC: one VPC three tiers, secrets as Secrets Manager references, never values.
  • A deterministic PreToolUse hook that blocks any plaintext secret in IaC.
  • A validation suite that asserts the migration, not just smoke tests.

The Gremlins: Couplings C1 to C5

  • C1 nginx /api proxy → CloudFront /api/* behavior to the ALB; same-origin, no CORS.
  • C2 one shared Postgres, batch reads app schema → keep a single RDS, two schemas, three roles.
  • C3 batch needs app tables first → cutover order + fail-fast and EventBridge retry.
  • C4 five teams connect directly → point them at the read replica, SG-scoped to analyst CIDR.
  • C5 24/7 daemon just to fire a 2am cron → schedule moves to EventBridge, task runs and exits.

How We Used Claude Code

  • Per-workload CLAUDE.md — the right guidance per folder.
  • Deterministic PreToolUse hook vs. probabilistic prompt preference (ADR-0003).
  • An ADR for every meaningful call, numbered in /decisions.
  • Cutover order and rollback designed in Plan Mode for the 4am call.
